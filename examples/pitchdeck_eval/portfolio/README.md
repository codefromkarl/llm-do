# Portfolio Directory

This directory contains all evaluated companies, one folder per company.

## Structure

```
portfolio/
├── CompanyA/
│   ├── CompanyA-Deck.pdf
│   ├── CompanyA-Evaluation.md
│   └── CompanyA-Evaluation-Log.md
├── CompanyB/
│   ├── CompanyB-Deck.pdf
│   ├── CompanyB-Evaluation.md
│   └── CompanyB-Evaluation-Log.md
└── ...
```

## File Types

### Pitchdecks (`*-Deck.pdf`)
- Original pitchdeck PDF
- Filename normalized (spaces removed, etc.)

### Evaluations (`*-Evaluation.md`)
- Comprehensive evaluation following framework
- Structured assessment with recommendations
- Generated by AI following `framework/eval_pitchdeck.md`

### Evaluation Logs (`*-Evaluation-Log.md`)
- Brief summary of each evaluation
- Includes date, evaluator, key findings
- Running history if company re-evaluated

### Additional Files
- `questions-to-founder.md` - Follow-up questions
- `due-diligence-notes.md` - DD findings
- `follow-up.md` - Communication with founders
- `*-analysis.md` - Additional analysis

## Workflow

1. **After evaluation**: Review AI-generated evaluation, add your notes
2. **Team collaboration**: Others can read, comment, add files
3. **Version control**: Git tracks all changes and additions
4. **Follow-ups**: Add new files to company folder as deal progresses

## Privacy

**IMPORTANT**: This contains confidential information.

- Keep repository private
- Don't share evaluations externally
- Consider adding `*.pdf` to `.gitignore` if pitchdecks are confidential
- Be mindful of what you commit

## Portfolio Summary

Track your overall portfolio stats by reviewing:
- Total companies evaluated
- Pass/proceed ratios
- Sector distribution
- Common rejection reasons

Consider creating a `portfolio-stats.md` file to track metrics over time.
